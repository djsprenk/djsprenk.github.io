Add an entry to CHANGELOG.md for the work just completed.

Follow the Keep a Changelog format (https://keepachangelog.com/en/1.1.0/):

- Place new entries under `## [Unreleased]`
- Use the appropriate subsection(s): `### Added`, `### Changed`, `### Fixed`,
  `### Removed`
- Each entry is a single bullet:
  `- short description of what changed and why it matters`
- Do not create a new version block — that happens at release time
- Today's date is available via the currentDate context if needed for a new
  release

If the user provides a description of the changes, use that. Otherwise summarise
the work from the current conversation.
